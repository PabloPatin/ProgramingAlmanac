# Настройка коммутатора
## Последовательность загрузки
1) Загрузка программы самопроверки (POST) из [[ПЗУ]]
2) Запуск начального загрузчика (boot loader)
3) Загрузчик инициализирует регистры ЦП для правильного отображения физической памяти
4) Загрузчик запускает файловую систему флэш-памяти
5) Загрузчик находит образ операционной системы IOS, загружает её в оперативную память и передаёт ей управление
### Выбор образа системы
`boot system flash:*путь_до_образа*`
## Индикаторы коммутатора
- SYST - ли система. зелёный - ОК; жёлтый - работает с перебоями; не горит - всё плохо.
- RPS - резервный источник питания. зелёный - ОК; зелёный мигает - система не доступна, питает другое устройство; жёлтый - режим ожидания или неисправность; мигает жёлтым - питание от резервного источника.
- STAT - режим статуса порта.
- DUPLX - дуплекс, если зелёный.
- SPEED - скорость. не горит - 10 Мб/c; зелёный - 100 Мб/с; мигает зелёным - 1000 Мб/с.
- PoE - мигает жёлтым - режим PoE не выбран; зелёный - режим PoE; не горит - PoE выключен; мигает зелёным и жёлтым - в PoE отказано.
## Доступ к управлению коммутатором
Осуществляется по консольному кабелю или через SVI
Для того, чтобы настроить SVI - нужно подключиться к консоли, задать одному из VLAN интерфейсов коммутатора IP адрес и протокол транспорта (SSH/Telnet)
## Настройка портов коммутатора
Задать режим дуплекса и скорость порта можно командами `duplex` и `speed` соответственно.
Для автоматического определения кабеля (прямой/перекрёстный) использовать команду `mdix auto`
## Команды проверки коммутатора
- `show interfaces *interface_id*`
- `show startup-config/running-config`
- `show flash`
- `show version`
- `show history` - история введённых команд
- `show ip[v6] interface {*interface_id* | brief}`
- `show mac address-table`
## Конфигурация SSH
1) `ip domain-name *domain_name*`
2) `crypto key generate rsa`
3) `username *name* secret *password*`
4) ```
   line vty 0 15
	   transport input ssh
	   login local
   ```
5) `ip ssh version 2`
