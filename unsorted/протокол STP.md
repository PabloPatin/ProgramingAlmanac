Протокол связующего древа. Помогает избежать петель при коммутации на уровне 2.
# Принцип работы
## Выбор корневого моста (коммутатора)
Опирается на показатель Bridge Priority
По умолчанию 32768
Значения в диапазоне 0-61440 с шагом 4096
Чем меньше, тем выше приоритет
К этому значению добавляется Extended System ID (для определения приоритета и VLAN BPDU)
В сценарии, когда BID одинаковый, выбирается 
## Выбор корневых портов
Корневой мост начинает рассылать BPDU, вычисляется стоимость пути (каждый раз при пересылке BPDU прибавляется стоимость порта)

| Скорость порта | Стоимость STP | Стоимость RSTP |
| -------------- | ------------- | -------------- |
| 10 Гб/с        | 2             | 2_000          |
| 1 Гб/с         | 4             | 20_000         |
| 100 Мб/с       | 19            | 200_000        |
| 10 Мб/с        | 100           | 2_000_000      |
Корневой порт ведёт к корневому мосту
## Выбор назначенных и альтернативных портов
Те порты, которые не были выбраны в качестве корневых, становятся назначенными
Если на порт, который не является корневым пришёл BPDU, то он становится заблокированным

Если есть два одинаковых по стоимости маршрута, то:
1) Выбирается путь у которого меньше BID отправителя
2) Выбирается путь с самым низким приоритетом порта отправителя
3) Выбирается самое низкое значение порта
## Таймеры STP
- hello timer - интервал между отправкой BPDU
- forward delay timer - таймер задержки пересылки (время, проводимое в состояниях прослушивания и обучения)
- max-age timer - максимальное время ожидание перед попыткой изменения топологии STP
Таким образом для классического STP проходят следующие этапы
Блокирующий режим --> Режим прослушивания (15 сек) --> Обучение (15 сек) --> Пересылка 
# Версии STP
- STP - классический стандарт IEEE 802.1D
- RSTP - ускоренный STP, стандарт IEEE 802.1W - 2004г
- PVST - per VLAN, для каждого VLAN можно настроить свой корневой мост
