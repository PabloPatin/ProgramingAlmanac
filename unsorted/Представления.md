объект базы данных, который является результатом SELECT запроса в момент обращения к представлению.

CREATE VIEW имя_представления AS select-запрос

С точки зрения теории множеств, представление - подмножество таблицы.

SELECT применим ко всем представлениям, в то время как UPDATE, INSERT, DELETE только для обновляемых представлений
## Ограничения
- Нельзя повесить триггер на представление
- Нельзя создать представление на основе временных таблиц
- Нельзя создать временное представление
- Нельзя создать из запроса с подзапросом в блоке FROM
- В определении представления нельзя использовать PREPARE
- При определении представления не используются системы и пользовательские переменные
- Обновляется только вместе с таблицей, к которой привязано представление
## Обновление представлений
1) Отсутствует DISTINCT
2) Отсутствует JOIN
3) Отсутствуют вычисляемые столбцы
4) WHERE не содержит вложенные запросы
5) В запросе не должно быть GROUP BY и HAVING
