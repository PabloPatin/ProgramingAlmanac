# Первая нормальная форма (1NF)
Все атрибуты таблицы должны быть атомарны
В каждом столбце содержится только одно значение (без перечислений)
## Пример нарушения
TABLE people

| id  | name    | value     |
| --- | ------- | --------- |
| 1   | Василий | 1, 2, 3   |
| 2   | Фёдор   | 3, 4, 100 |
## Пример исправления
TABLE people

| id  | name    |
| --- | ------- |
| 1   | Василий |
| 2   | Фёдор   |
TABLE values

| id  | value |
| --- | ----- |
| 1   | 1     |
| 1   | 2     |
| 1   | 3     |
| 2   | 3     |
| 2   | 4     |
| 2   | 100   |
# Вторая нормальная форма
Таблица находится в первой нормальной форме
Все не ключевые атрибуты зависят только от первичного ключа
## Пример нарушения
TABLE sales

| Номер заказа | Клиент | Товар | Отдел  |
| ------------ | ------ | ----- | ------ |
| 1            | Иванов | стул  | мебель |
| 2            | Петров | стол  | мебель |
## Пример исправления
TABLE sales

| Номер заказа | Клиент | Товар |
| ------------ | ------ | ----- |
| 1            | Иванов | стул  |
| 2            | Петров | стол  |
TABLE products

| Отдел  | Товар |
| ------ | ----- |
| мебель | стул  |
| мебель | стол  |
# Третья нормальная форма
Отсутствие транзитивных функциональных зависимостей
Все неключевые атрибуты напрямую зависят от колючевых

# Нормальная форма Бойса-Кодда
Для любой нетривиальной зависимости X-->Y множество X - суперключ
Часть составного первичного ключа не должна зависеть от неключевого столбца
# Четвёртая нормальная форма
В таблице отсутствуют многозначные зависимости
Если А-->B и С-->B, и при этом А и С независимы, то неправильно A, B и C держать в одной таблице
# Пятая нормальная форма
Требует, чтобы каждая нетривиальная зависимость соединения в таблице определялась потенциальным ключом (ключами) этой таблицы.
Максимальное разбиение при котором можно воссоздать оригинал
# Доменно ключевая форма
Для каждого поля есть ограничения домена, которые должны быть соблюдены
# Шестая нормальная форма
Невозможна дальнейшая декомпозиция без потерь
