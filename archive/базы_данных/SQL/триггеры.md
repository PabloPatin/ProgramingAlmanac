Триггеры - хранимые процедуры, которые автоматически вызываются после определённых событий. Часто используются для поддержки целостности данных.
Триггер всегда привязан к таблице. Ограничения таблицы могут препятствовать работе триггера, что отменяет его выполнение.

Важно, чтобы триггеры не вызывали срабатывания друг друга, иначе может произойти зацикливание

- Триггер не может быть вызван напрямую. 
- У триггера нет входных и выходных значений.
- Может иметь до 16 уровней вложенности

Классификация триггеров по области действия:
- Ограничения домена
- Ограничения атрибута
- Ограничения кортежа
- Ограничения отношения
- Ограничения базы данных

Триггер всегда замедляет работу базы данных, так что по возможности лучше обходиться без них.
Тем не менее в следующих случаях триггеры могут быть крайне полезными:
1. Поддержка сложных ограничений
2. Аудит изменения данных
3. Генерация идентификаторов строк
4. Заполнения вычисляемых колонок
5. Репликация данных

# Синтаксис
	DELIMITER |
	CREATE TRIGGER [TRIGGER_NAME]
	...

У триггеров есть псевдонимы NEW и OLD
NEW использует новые значения кортежа, а OLD - старые
