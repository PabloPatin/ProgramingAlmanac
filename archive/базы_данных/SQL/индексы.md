Индексы - объекты базы данных, позволяющие оптимизировать поиск информации.
Строки в таблицах хранятся в неупорядоченном виде по умолчанию для ускорения поиск используется индекс.
На основе данных, содержащихся в конкретной строке таблицы формируется значение элемента индекса, соответствующего этой строке.
Индекс может определять строку как однозначно, так и неоднозначно.

Пример: при поиске по имени группы, логично проиндексировать столбец имени группы для ускоренного поиска (чтобы не использовать всегда id)

Первичный ключ по умолчанию ВСЕГДА является индексом.
Внешний ключ НЕ ЯВЛЯЕТСЯ индексом по умолчанию
Процесс объединения таблиц ускорится, если индексировать внешний ключ таблицы.

Индекс упорядочен.
Полный перебор таблицы заменяется нахождением индекса и перехода по прямой ссылке на строку таблицы.

## Синтаксис
### Простой индекс
	CREATE INDEX name ON table_name (column_name);
### Составной индекс
	CREATE INDEX name ON table_name (column_name1, column_name2, ...);
### Уникальный индекс
	CREATE UNIQUE INDEX name ON table_name (column_name);
NULL тоже уникальное значение
Ограничение столбца UNIQUE не создаёт индекса по умолчанию, его надо индексировать именно так.
## Частичные индексы (только PostgreSQL)
	CREATE INDEX name ON table_name (column_name1) WHERE условие;
Не слишком эффективно в случае, если индексированный столбец часто изменяется (повторно индексирует после изменения столбца)
# Кластерный индекс
Тип индекса с древовидной структурой. Каждый уровень дерева представляет собой индексные страницы, а конечные страницы содержат реальные данные строк.
Влияет на порядок физического расположения записей.
(B-tree, hash)
# Некластерный индекс
То же, что и кластерный, но не влияет на физическое расположение записей.
(B-tree, hash)
