Порт, через который пакет приходит на коммутатор - **входной**
Порт, через который коммутатор отправляет пакет - **выходной**
>[!note]
>Кадр Ethernet никогда не будет перенаправлен на тот же порт, с которого он был получен.
# Таблица MAC-адресов

| MAC-адрес         | порт |
| ----------------- | ---- |
| aa-aa-aa-aa-aa-aa | Fe1  |
Хранится в ассоциативной памяти (CAM)
Коммутатор пополняет эту таблицу только MAC адресами источника для каждого кадра
Адрес будет удалён из таблицы если таймер (часто 5 минут) истёк и ни одного кадра не было получено от устройства
## Одноадресная рассылка
Если адрес есть в таблице - пересылка по нужному порту
Иначе - пересылка на все порты кроме входящего
# Коммутация с промежуточным хранением (store-and-forward)
Коммутатор получает кадр целиком
Сверяет контрольную сумму (FCS - frame check sequence)
Принимает решение о пересылке
# Сквозная коммутация (cut-through)
Коммутатор начинает пересылку сразу, как только получил MAC-адрес назначения
# Коммутационные домены
## Домен коллизий
Область, в которой может возникать коллизия между кадрами
В полнодуплексном режиме не существует
В полудуплексном - каждое соединение является доменом коллизии
Каждый порт коммутатора (в полудуплексном режиме) - отдельный домен коллизии
Концентратор, в отличии от коммутатора объединяет все порты в один домен коллизии
## Широковещательный домен
Область которую охватывает широковещательная рассылка
Широковещательные домены делит маршрутизатор, а коммутаторы и концентраторы объединяют
