# Сетевые операционные системы
При помощи сетевой операционной системы на базе [[CLI]], такой как [[Cisco IOS]] на коммутаторе или маршрутизаторе, сетевой специалист может выполнять следующие задачи.
- Запускать сетевые программы на базе CLI, используя клавиатуру.
- Вводить текст и текстовые команды с клавиатуры.
- Просматривать выходные данные на экране монитора.
# Способы доступа
| метод                  | описание                                                                                                                                                                                                                                                                                                                                   |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Консоль                | Физический порт управления, обеспечивающий внеполосный доступ к устройству. Используется исключительно в целях технического обслуживания устройства. <br>Доступен без настройки сетевых сервисов. Подходит для начально настройки сетевого устройства.<br>Для подключения требуется компьютер с программой эмуляции терминала.             |
| [[SSH]] (Secure Shell) | Метод, позволяющий удаленно установить защищенное подключение CLI через виртуальный интерфейс по сети.<br>В отличие от консольного подключения для SSH-подключений на устройстве должны быть активны сетевые службы, включая активный интерфейс с настроенным адресом.                                                                     |
| [[Telnet]]             | Незащищенный протокол, позволяющий удаленно начать сеанс CLI через виртуальный интерфейс по сети.<br>В отличие от SSH, Telnet не обеспечивает безопасное зашифрованное соединение и должен использоваться только в лабораторной среде. Данные для аутентификации пользователя, пароли и команды передаются по сети в виде простого текста. |

# Навигация по [[Cisco IOS]]
## Командные режимы
### Пользовательский режим EXEC
Доступен по умолчанию при входе в сетевое устройство.

Режим с ограниченными возможностями, но он удобен для базовых операций. 
В пользовательском режиме доступно только ограниченное число основных команд мониторинга, но невозможно выполнять какие-либо команды, которые могут изменить конфигурацию устройства.

Пользовательский режим EXEC можно определить по командной строке CLI, оканчивающейся символом «>».
	Switch>
	Router> 
### Привилегированный режим EXEC
Для перехода в привилегированный режим необходимо выполнить команду ***enable***, после чего зачастую необходимо ввести пароль.

Этот режим должен использовать сетевой администратор для выполнения команд настройки.
Режимы конфигурации более высокого уровня, например, режим глобальной конфигурации, доступны только из привилегированного режима EXEC. 

Привилегированный режим EXEC можно определить по командной строке, оканчивающейся символом # .
	Switch#
	Router#
### Вложенные режимы конфигурации
#### Режим глобальной конфигурации
Доступен только из пользовательского режима EXEC

Переход в него осуществляется командой 
***configure terminal***

В режиме глобальной конфигурации выполняются изменения конфигурации CLI, влияющие на работу устройства в целом.
	Switch(config)#
	Router(config)#
#### Режим настройки линии
Предназначен для настройки доступа через одну из физических или виртуальных линий (консоль, SSH, Telnet или AUX).

Вход осуществляется из режима глобальной конфигурации командой 
***line \[тип линии\] \[номер линии\]***

Запрос по умолчанию
	Switch(config-line)#
	Router(config-line)#
#### Режим настройки интерфейса
Предназначен для настройки порта коммутатора или сетевого интерфейса маршрутизатора.

Вход осуществляется из режима глобальной конфигурации командой 
***interface \[тип интерфейса\] \[номер интерфейса\]***

Запрос по умолчанию
	Switch(config-if)#
	Router(config-if)#

## Команды для возврата из режимов конфигурации
***exit*** - переносит на один уровень выше
***end*** - переносит из любого режима конфигурации в привилегированный режим EXEC
***ctrl + z*** - то же что и end
***disable*** - переход в пользовательский режим EXEC
