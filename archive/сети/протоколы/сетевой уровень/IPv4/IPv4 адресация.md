# IP-адрес
Уникальный адрес устройства в сети, построенной на основе стека протоколов [[TCP/IP]]
IP-адрес определяет [[хост]] и сеть, к которой он подключен.
Занимает 32 бита.
Записывается в виде четырёх [[октет|октетов]]
192.168.0.0
## Частные и публичные адреса
IPv4 адреса бывают двух типов: публичные и частные. Публичные адреса используются для выхода в **[[интернет]]**, а частные - в **локальных сетях**.

Частные адреса могут быть назначены любым устройством локальной сети её администратором.

Публичный IP должен быть выдан **провайдером**. Провайдер получает публичные адреса от **локального Интернет регистратора** (LIR), он - от **регионального Интернет регистратора** (RIR). Региональному регистратору IP-адреса выдаёт организация **[[IANA]]**. Контролирует действие организации **[[IANA]]** компания **[[ICANN]]**.

Вы  -> провайдер -> LIR -> RIR -> IANA -> ICANN 
# Маска подсети
Указывает на то, какие биты в IP-адресе идентифицируют подсеть, в которой находится адресуемое устройство
Нулями в битовом представлении маски обозначается узловая часть адреса (она отвечает за адресацию устройства в подсети)

**Вот как это работает**
    IP: 192.168.0.115
    Mask: 255.255.255.0
    Subnet: 192.168.0.0
    Host: 0.0.0.115
## Префиксная форма
Для упрощения записи, маску иногда пишут в сокращённой префиксной форме сразу после IP-адреса
**Такая запись адреса:**
    192.168.128.0/20
**Означает следующее**
    IP: 192.168.128.0
    Mask: 255.255.240.0
Число после знака '/' (префикс) означает количество бит в маске, равных 1
# Классы сетей
## Класс А
Адреса: 1.0.0.0 - 126.0.0.0
Маска: 255.0.0.0
Используется для особо крупных сетей
Частные адреса: 10.0.0.0/8
## Класс B
Адреса: 128.0.0.0 - 191.255.0.0
Маска: 255.255.0.0
Для средних по размеру (часто региональных) сетей
Частные адреса: 172.16.0.0/12
## Класс C
Адреса: 192.0.0.0 - 223.255.255.0
Маска: 255.255.255.0
Для небольших сетей
Частные адреса: 192.168.0.0/16
## Класс D
Адреса: 224.0.0.0 - 239.255.255.255
Маска: 255.255.255.255
Для многоадресных рассылок
## Класс Е
Адреса: 240.0.0.0 - 247.255.255.255
Маска: 255.255.255.255
Используются для экспериментов

# Адресация в локальной сети
Для примера возьмём сеть 172.16.13.0/24
172.16.13.0 - адрес сети, он зарезервирован и не может быть назначен [[хост|хосту]].
172.16.13.1 - адрес первого устройства в сети (чаще всего используется в качестве шлюза по умолчанию, но это не обязательно)
172.16.13.254 - адрес последнего устройства в сети
172.16.13.255 - широковещательный адрес ([[broadcast]])
Таким образом сеть может адресовать 2^(32-mask)-2 узлов, один из которых будет назначен в качестве шлюза по умолчанию.
# Зарезервированные IP-адреса

1) Loopback адрес: 127.0.0.0 - 127.255.255.255 
	Он используется для самодиагностики компьютера.

2) [[APIPA]] (Automatic Private IP Addressing): 169.254.0.0 - 169.254.255.255
	Компьютер может взять себе адрес из этого диапазона автоматически том случае, если [[DHCP]] не настроен в локальной сети. Это помогает ему начать работать с сетью.
