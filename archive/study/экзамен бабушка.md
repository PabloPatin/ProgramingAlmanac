# Основные определения
Алфавит - множество символов, из которых состоят программы
Синтаксис - совокупность правил, для построения программ на ЯП
Семантика - совокупность правил интерпретации предложений и конструкций ЯП
Грамматика - механизм конечного представления языка
# Архитектура Фон Неймана
## Основные компоненты
1. **Процессор (CPU)**
    - арифметико-логическое устройство (АЛУ);
    - устройство управления.
2. **Память**
    - единая для **данных и программ**.
3. **Устройства ввода/вывода**
4. **Шина данных**
    - передаёт данные, адреса и управляющие сигналы.
## Ключевые принципы
### 1. Хранимая программа
Программа хранится в памяти **наравне с данными** и может обрабатываться как данные.
### 2. Последовательное выполнение
Инструкции выполняются **строго по порядку** (если не задан переход).
### 3. Общая память
Данные и команды используют **один и тот же канал доступа**.
### 4. Изменяемое состояние
Вычисления — это последовательность инструкций, **меняющих содержимое памяти**.
## Цикл выполнения команды
1. Считать команду из памяти
2. Декодировать
3. Выполнить
4. Перейти к следующей
## Узкое место Фон Неймана
**Von Neumann bottleneck** — ограничение производительности из-за:
- общей шины для данных и команд;
- невозможности одновременно читать инструкцию и данные.

# Формальные грамматики
**Определение грамматики G=(N,T,S,P)**
N - множество нетерминальных символов
T - множество терминальных символов
S - исходный символ
P - множество продукций

**Язык**, порождаемый формальной грамматикой - множество цепочек, состоящих только из терминальных символов, выводимых из начального символа за конечное число шагов
## Дерево синтаксического анализа
В корне - начальный символ
В промежуточных вершинах - произвольные нетерминальные символы
В листьях - произвольные терминальные символы
## Иерархия грамматик Хомского
4 типа грамматик
Накладывает ограничения на правила вывода
### 0 - тип (Свободные грамматики/неограниченные)
Эквивалентна машине Тьюринга
#### Форма правил
- **α → β**,
	где α содержит хотя бы один нетерминал.
### 1 - тип (Контекстно-зависимые/неукорачивающие)
Правило зависит от **контекста**, в котором стоит символ.
Соответствует **линейно-ограниченным автоматам**.
#### Форма правил
- αAβ → αγβ, где **|γ| ≥ 1**
- Если нет правил вида αAβ → αεβ, то неукорачивающая
### 2 - тип (Контекстно-свободные/КС)
Правило НЕ зависит от **контекста**, в котором стоит символ.
Позволяют описывать **вложенные структуры**.
Соответствуют **магазинным автоматам** (pushdown automata)
#### Форма правил
- **A → γ**,  
    где **A** — один нетерминал, **γ** — строка терминалов и нетерминалов.
#### S - грамматика
Грамматика называется **S-грамматикой**, если для каждого нетерминала **A** и любых двух различных правил справедливо, что
При A → αγ | βγ 
$\alpha,\beta \in T, \alpha \neq \beta, |\gamma| \ge 0$
### 3 - тип (Регулярные/автоматные)
Описывают **регулярные языки** — самые простые.  
Не умеют считать и хранить вложенность.
Эквивалентны **конечным автоматам** и **регулярным выражениям**
#### Форма правил
- **A → aB** - праволинейные
- **A → Bа** - леволинейные
- **A → a**  
    (лево- или праволинейные, но не смешанные)
## Нотация Бэкуса-Наура (BNF)
Синтаксис задаётся **набором правил вывода**, которые показывают, как из стартового символа можно получить корректную строку языка.
### ## Основные обозначения
- <нетерминал> - нетерминалы внутри угловых скобок
- \[необязательный\] - необязательные элементы внутри квадратных скобок
- {повтор >= 0} - внутри фигурных скобок элементы, которые повторяются 0 и более раз
- (группа) - для группировки используются круглые скобки
- + - повтор 1 и более раз
- ? - включение 0 или 1 раз, аналог \[\]
- альт. 1 | альт. 2  - выбрать один из вариантов
- ::= -аналоги присвоения (это есть)
